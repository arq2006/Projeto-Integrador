Projeto Integrador 2026/1 - ADS 1º período

Velaris PetCare

MVP simples e navegável para divulgação de serviços pet comunitários, criado como recorte do tema Conecta Comunidade.

Slogan

Conectando cuidado e confiança para seu pet.

Ideia central

Reunir em um só lugar informações básicas de pet shops, clínicas, vacinação, hotel pet, taxi dog e produtos pet.

Tecnologias

React, Vite, React Router DOM, CSS/Tailwind e dados mockados.

Tipo de entrega

Protótipo front-end, sem backend, feito para apresentação em banca.

Escopo

Simples, direto e compatível com primeiro período.

1. Problema e solução

Contexto comunitário

Problema social

Muitos serviços pequenos são divulgados de forma espalhada em grupos, redes sociais, cartazes e indicações. Isso dificulta para o tutor encontrar informações confiáveis de bairro, contato e horário.

Solução proposta

Um site simples para listar serviços pet comunitários, com busca, filtros por categoria e bairro, página de detalhes, formulário de sugestão e painel administrativo apenas conceitual.

Público-alvo

  • Tutores de pets que procuram serviços próximos.
  • Pet shops, clínicas veterinárias, lojas e prestadores locais.
  • Moradores que querem sugerir novos serviços para a comunidade.

Objetivo geral

Criar um MVP funcional para centralizar serviços pet locais e facilitar o acesso da população a informações básicas e contatos.

3. Algoritmos e Programação

Lógica usada no protótipo

Conceitos aplicados

  • Listas/vetores para guardar serviços, categorias e sugestões.
  • Condicionais para verificar filtros preenchidos.
  • Repetição com map para mostrar cards na tela.
  • Filter para selecionar serviços compatíveis com a busca.
  • Estado do React para controlar busca e formulários.

Pseudocódigo da busca

Começar com todos os serviços. Se houver texto digitado, procurar no nome, categoria, descrição e palavras-chave. Se houver bairro, manter apenas o bairro escolhido. Se houver categoria, manter apenas serviços daquela categoria. Por fim, exibir os cards.

Arquivos relacionados

  • Services.jsx - busca e filtros.
  • SuggestService.jsx - formulário e sucesso fake.
  • Agendamento.jsx - agendamento simulado.
  • src/data - dados mockados.
4. Engenharia de Software

Requisitos, casos de uso e organização

Requisitos funcionais

  • Listar serviços por categoria.
  • Buscar por nome, bairro e palavra-chave.
  • Mostrar detalhes do serviço.
  • Permitir sugestão de novo serviço.
  • Mostrar painel administrativo conceitual.

Requisitos não funcionais

  • Interface simples e responsiva.
  • Dados mockados, sem backend.
  • Navegação com React Router DOM.
  • Visual coerente com a identidade Velaris.

Casos de uso resumidos

  • O tutor busca um serviço pet.
  • O tutor abre os detalhes do estabelecimento.
  • O tutor simula um agendamento.
  • O morador sugere um novo serviço.
  • O administrador visualiza dados simulados.

6. Governança de TI e Empreendedorismo

Viabilidade, impacto e Risk IT

Viabilidade

  • Projeto pequeno, feito em front-end.
  • Não depende de servidor ou banco de dados nesta etapa.
  • Pode evoluir futuramente para sistema real.

Impacto na comunidade

  • Facilita encontrar serviços pet locais.
  • Dá visibilidade a pequenos negócios.
  • Ajuda tutores a comparar contatos, horários e categorias.

Riscos e ações preventivas

Risco Ação preventiva
Dados incorretos de estabelecimentos Revisão administrativa antes de publicar.
Informações falsas Fila de revisão para sugestões da comunidade.
Exposição de dados Coletar apenas informações necessárias.
Sistema fora do ar no futuro Usar hospedagem confiável e backup.

9. Conclusão

Resultado do protótipo

O Velaris PetCare está coerente com o PDF do Projeto Integrador porque entrega um MVP simples, funcional e conectado ao tema comunitário. O projeto não tenta ser um sistema completo: ele funciona como esboço navegável para demonstrar a solução, os requisitos e a integração com as disciplinas.
